## Notes

- **All 24 visual sections matched to library designs** — zero custom designs needed
- **Service deep-dives paired for layout variety:** S07+S08 use stacked blocks, S09+S10 use accordion, S11+S12 return to stacked blocks — creates visual rhythm
- **Modality Selection Framework (S13)** uses transformation-phase-cards — the Problem/Solution/Outcome tri-colour structure maps perfectly to the VR/AR/Desktop/Case Study modality comparison
- **Background alternation:** hero (white) → definitional (white) → challenges (white+image) → stats (light-bg) → pillars (white) → services (light-bg) → deep-dives (alternating) → modality cards (white) → process (navy) → outcomes (light-bg) → industries (white) → differentiators (white) → case studies (light) → testimonials (light-bg) → logo wall (navy) → FAQ (light-bg) → download (navy) → CTA (lime) → form (white) → connected (navy) → resources (white) → footer (dark navy)
- **FAQ section:** Library design supports 5 items; content has 6 — developer must add one extra accordion item
- **Industries section:** `industries-selectable-tiles` chosen because Healthcare (Industry 1) has "4 feature bullets · 4 mini stat cards" — the interactive tiles variant supports this level of detail
- **Content file uses 27 section numbers** (S01-S27) but the page flow has 24 entries because service deep-dives are paired (S07+S08 → 1 entry, S09+S10 → 1 entry, S11+S12 → 1 entry)
